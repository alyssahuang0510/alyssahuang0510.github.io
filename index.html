Redesigning the X-ray system that inspects every board before it ships

改版後總編程時間減少——以兩塊量產板量測（295→186、386→237 分鐘）

TRI’s flagship 3D AXI software detects solder-joint defects on every PCB leaving a production line — operated in shifts, where a confusing flow costs real downtime. I led the full redesign from a legacy 2D AOI interface into a 3D-AXI-native workflow, then owned it for six years as the sole UI/UX designer. The redesign cut total programming time by ~38% — independently measured by FAE teams on two production boards (a 4,160-component board: 295→186 min; a 3,495-component board: 386→237 min), with the heaviest gains in the Fine Tune and Library steps.

德律的旗艦 3D AXI 軟體負責檢測產線上每一片 PCB 的焊點缺陷——以輪班方式操作，一個令人困惑的流程就是真實的停機損失。我主導了從舊版 2D AOI 介面到 3D-AXI 原生工作流程的完整重新設計，並以唯一 UI/UX 設計師的身分負責了六年。改版讓總編程時間減少約 38%——由 FAE 團隊在兩塊量產板上獨立量測（4,160 顆零件的板：295→186 分；3,495 顆零件的板：386→237 分），改善最大的是 Fine Tune 與 Library 兩個步驟。

  • Fine Tune −51% (158→77 min), Library −39% (124→76 min) — the two slowest steps. Collapsing F3/F4 into one Fine Tune page and replacing manual height calculation with mouse-wheel adjustment is what drove most of the saving.
  • One standardized interface across the entire AXI line (SIII / LL / TL / F3D / FB / HP / SV families) — consistency that survives every new machine.
  • Owned programming, alignment, GR&R, and multi-language deployment for global major accounts (automotive, server, semiconductor).
  • Worked directly with algorithm & mechanical teams; C# / MFC literacy made spec arguments shorter and feasibility calls faster.
  • Fine Tune −51%（158→77 分）、Library −39%（124→76 分）——兩個最慢的步驟。把 F3／F4 整併進單一 Fine Tune 頁、以滑鼠滾輪取代手動計算高度，是時間縮短的主因。
  • 單一標準化介面套用於整條 AXI 產品線（SIII / LL / TL / F3D / FB / HP / SV 系列）——每出一台新機都能維持一致性。
  • 獨力負責編程、定位、GR&R 與多國語系部署，服務汽車、伺服器、半導體等全球大客戶。
  • 直接與演算法與機構團隊協作；C# / MFC 的底子讓規格討論更短、可行性判斷更快。

I owned its UX from zero to launch: product positioning, competitive teardown, information architecture, result hierarchy, and the integration boundary with the rest of MyExpert (Chat, Knowledge Hub, ExpertMeet).

File Search 將本機與雲端檔案整合在單一查詢之後，從 2026 年起預載於華碩商用筆電、AiO 與桌機。我從零到上線負責它的 UX：產品定位、競品拆解、資訊架構、搜尋結果階層，以及與 MyExpert 其他模組（Chat、Knowledge Hub、ExpertMeet）的整合邊界。

  • Reconciled two mental models in one result list — file-manager behaviour (browse, sort) and search behaviour (rank, snippet) — without making either feel bolted on.
  • Scoped it as a peer app to Chat and Knowledge Hub, defending product-level boundaries against feature-creep pressure.
  • Mapped the AI integration surface so File Search can graduate into AI workflows without re-architecting its IA.
  • 在同一份結果清單中調和兩種心智模型——檔案管理（瀏覽、排序）與搜尋（排名、摘要）——讓兩者都不像勉強拼裝。
  • 把它定位為與 Chat、Knowledge Hub 並列的 App，在功能擴張的壓力下守住產品級邊界。
  • 梳理 AI 整合介面，讓 File Search 能升級進 AI 工作流程，而不必重構其資訊架構。

I inherited a queue of high-stakes features touching enterprise device fleets, where a confusing flow means real downtime on real machines. My job: make cryptographic and policy complexity legible to non-developer IT admins.

ASUS Config Manager 擴充了微軟 SCCM——加入 BIOS 設定、加速更新，並透過 Intune/DFCI 進行遠端管理。我接手了一系列牽動企業裝置群的高風險功能，在這裡，一個令人困惑的流程就代表真實機器的停機。我的工作：把密碼學與政策的複雜度，翻譯成非開發背景的 IT 管理者也能讀懂的介面。

  • Encrypted BIOS deployment & Silent Update — surfaced security posture and update state without burying admins in jargon.

  • BIOS Asset Tag & JSON deployment / response — built a structured input flow admins can configure confidently.
  • 加密 BIOS 派送與靜默更新——把安全狀態與更新狀態清楚呈現，而不讓管理者淹沒在術語裡。
  • Driver Command Policy——設計版本偵測、嚴重度篩選與可自訂的更新頻率。

跨兩條產品線、從零設計的平台功能

When Android ships a platform capability, every OEM must design the UX above it — settings IA, entry points, toggle behaviour, and the conflicts between features that can’t coexist. I led from-zero UX for Predictive Back, Edge-to-edge, Circle to Search, Auto Rotate, default Gemini assistant, and gaming-phone gyroscope on ROG and Zenfone. The hardest work isn’t any single feature — it’s the system view.

當 Android 推出一個平台能力，每家 OEM 都得設計它之上的 UX——設定的資訊架構、入口、開關行為，以及無法並存的功能之間的衝突。我在 ROG 與 Zenfone 上，從零主導了 Predictive Back、Edge-to-edge、Circle to Search、Auto Rotate、預設 Gemini 助理，以及電競手機陀螺儀的 UX。最難的不是任何單一功能——而是系統視角。

  • Conflict resolution — for mutually exclusive features, defined the rule set, the resolution UI, and the recovery path.
  • Cross-product differentiation — ROG (gaming, tunable) vs Zenfone (general); diverged the IA only where it earned its keep.
  • Also redesigned and stabilized Screen Recording after takeover.
  • 衝突處理——針對互斥功能，定義規則集、衝突解決的 UI 與復原路徑。
  • 跨產品線差異化——ROG（電競、可調校）對比 Zenfone（一般消費）；只在值得的地方讓資訊架構分流。
  • 另接手並重新設計、穩定了螢幕錄影功能。

[03]   About關於

A designer who shipped software before she shipped screens.

在設計介面之前，先交付過軟體的設計師。

I’m a pure UX designer working on tool-shaped products — the kind operators, IT admins, and developers depend on daily. My undergraduate degree is in Information Management; my master’s is in Interaction Design. That dual root means I read engineering tickets the way I read research notes.

我是專注於工具型產品的純 UX 設計師——那種產線操作員、IT 管理者與開發者每天依賴的產品。我的學士是資訊管理，碩士是互動設計。這雙重根基讓我讀工程票券，就像讀研究筆記一樣自然。

I spent six years shipping C# / MFC desktop software and built an AR rehabilitation game in Unity for my thesis. Code isn’t my output anymore, but it shapes how I think about feasibility, system constraints, and exactly what to ask engineers for — which is why cross-functional teams move faster with me in the room.

我花了六年交付 C# / MFC 桌面軟體，碩士論文則用 Unity 做了一款 AR 復健遊戲。寫程式不再是我的產出，但它形塑了我對可行性、系統限制，以及該向工程師要什麼的判斷——這也是為什麼有我在的跨職能團隊跑得更快。

The tools I gravitate toward are unglamorous: dense data, expert users, long task sessions, conflicting modes. My job is to make those legible — and increasingly, to build the tools my own design team plans with.

我偏好的工具都不華麗：高密度資料、專家使用者、長時間的任務、彼此衝突的模式。我的工作是讓這些變得清晰可讀——而且越來越多時候，是親手打造設計團隊用來規劃的工具。
